Gestion Bar

  • .Net
  • Razor
  • C#

Description

Ce projet, réalisé dans le cadre de mes études à l'Université de Picardie Jules Verne, avait pour objectif de simuler une partie du système d'information (SI) d'un bar souhaitant dématérialiser la gestion des commandes des consommateurs. Le projet impliquait trois acteurs principaux : le serveur, le barman et le caissier. Chaque acteur disposait d'une solution numérique spécifique pour améliorer ses tâches.


Les fonctionnalités principales de l'application incluaient :


  • Application de prise de commande : Le serveur utilise une tablette pour prendre les commandes des consommateurs à une table numérotée. Une fois la commande validée, elle est envoyée au barman pour préparation. Le serveur peut suivre l'état des tables et des commandes en temps réel.
  • Application de préparation de la commande : Le barman dispose d'une interface sur un grand écran tactile affichant les commandes prises par les serveurs et en cours de préparation. Une fois la commande prête, le barman la valide pour que le serveur puisse la récupérer.
  • Application de gestion financière : Le caissier encaisse le montant une fois que le serveur a indiqué que la commande est servie. Il peut gérer les paiements partagés pour une même commande et éditer des factures sur demande. Cette application inclut également le système de gestion financière de l'établissement et le système de gestion des stocks.

Le projet impliquait de développer trois clients applicatifs liés au même ensemble de processus métiers, offrant une solution numérique complète pour la gestion des commandes, la préparation des commandes et la gestion financière. Ce projet m'a permis de renforcer mes compétences en développement d'applications web et en gestion de processus métier dans un environnement de simulation réaliste.
